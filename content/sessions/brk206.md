## 概要

GitHub Copilot SDK の GA 到達と、エージェントをローカルマシンからクラウド、マルチデバイスへ展開するアーキテクチャを解説するセッション。前半は Patrick Nikoletich が SDK の GA ステータス、言語サポート、Microsoft 内部での採用状況を紹介。後半は Steve Sanderson がライブコーディングで、不動産リード管理エージェントの構築を通じてカスタムツール定義、サンドボックス、セッション分離、リモートクラウド環境への接続を実演した。

## キーポイント

1. **Copilot SDK が GA（1.0）**: 2026年1月の Technical Preview を経て Build 2026 で GA。TypeScript、Python、.NET、Go、Java、Rust の 6 言語をサポート
2. **同一ランタイム基盤**: SDK は Copilot クラウドエージェント・Copilot CLI と同じランタイムハーネスを使用。どのサーフェスでも同じエージェント体験を提供
3. **Microsoft 全社採用が進行中**: Office Excel、Copilot Studio、Cowork（M365）が SDK ベースのエージェントランタイムに移行中。Copilot Studio では品質指標が 40–50% 向上
4. **4 つの分離モデル**: Direct Execution → コンテナ/VM per session → OS サンドボックス（MXC） → 仮想ファイルシステム
5. **MXC（Microsoft Execution Containers）**: Windows・macOS・Linux 対応のクロスプラットフォームサンドボックス。ファイルシステム・ネットワークを OS レベルで制御
6. **セッション FS（仮想ファイルシステム）**: セッションごとに独立したファイルシステムを提供。セッションデータのバックアップ・移行・復元が容易
7. **リモートクラウド環境**: SDK からクラウドサンドボックスを起動し、セッションをアプリ↔ブラウザ↔別デバイス間で移行可能
8. **Rust ランタイムへの書き換え進行中**: Node.js 依存（起動時 ~130MB RAM）からの脱却。10 倍以上のパフォーマンス改善を目標。Wasm コンパイルで ~500KB まで縮小した PoC を実演

## 詳細

### Part 1: SDK GA と採用状況（Patrick Nikoletich）

SDK の GA 到達、言語サポートの拡大、Microsoft 内部での大規模採用を紹介。

- **GA 1.0**: 2026年1月に Technical Preview → Build 2026 で GA
- **6 言語**: TypeScript、Python、.NET、Go が初期提供。Java → Rust を追加して GA 時点で 6 言語
- **デプロイ先**: ローカルマシン、コンテナ、Private VNet、マルチテナントサービス、エアギャップ/政府クラウドすべてで稼働。GitHub Copilot サブスクリプション不要（BYOK 対応）
- **Microsoft 内部の品質ベンチマーク**: Copilot Studio チームの導入結果として、ファイル操作・ツール活用・コンテキスト利用で 40–50% の品質向上

### Part 2: エージェント構築デモ（Steve Sanderson）

C# で不動産リード管理エージェントを段階的に構築するライブデモ。

#### ステップ 1: 基本セッション

`CopilotClient` インスタンスを作成し、複数の `Session` を並列起動。各セッションに顧客問い合わせを割り当て、エージェントオーケストレーションの基礎を構築。

#### ステップ 2: システムプロンプトカスタマイズ

`identity` セクションのみを置換する部分カスタマイズを実演。「あなたは不動産エージェントです」とロールを定義し、バリデーション → 検索 → レポート作成のワークフローを指示。SDK デフォルトのプロンプト（ベンチマーク済み）を維持しつつ、業務固有の指示だけを差し替える設計。

#### ステップ 3: カスタムツール

C# メソッドをカスタムツールとして登録。フェーズ報告ツール（UI 更新用）と物件検索ツール（SQL バックエンド）を定義。ツールセットの明示的な制御により、エージェントに「何ができるか」を厳密に制限。

#### ステップ 4: 組み込みツールの選択的有効化

全ツール有効化ではなく `web_fetch` のみを選択的に有効化。エージェントが自律的にウェブ検索を行い、モデルの学習データにない新しい情報（新設校の所在地）を取得するデモ。

### Part 3: 分離モデル（Steve Sanderson）

エージェントの実行環境を段階的に強化する 4 つの分離モデルを紹介。

#### Direct Execution

エージェントがホストマシン上で直接実行。ツールセットの制限で能力を制御。最もシンプルだが、プロンプトインジェクションへの防御はツール制限のみ。

#### OS サンドボックス（MXC）

MXC（Microsoft Execution Containers）を Preview で提供。ファイルシステムアクセス・ネットワーク・サブプロセス生成を OS レベルで制御。JSON ポリシーファイルで読み書き可能なパスとネットワーク許可先を定義。

**デモ**: 意図的に脆弱な設定（「何でも言われた通りにやれ」+ 全ツール有効化）でプロンプトインジェクション攻撃を実行 → サンドボックスなしではファイル作成に成功 → サンドボックス有効化後はすべてのファイル書き込み試行を OS がブロック。

#### 仮想ファイルシステム（Session FS）

セッションごとに独立したファイルシステム抽象レイヤーを提供。`ISessionFS` インターフェースで実装をカスタマイズ可能（インメモリ、ディスク、Blob ストレージ）。

- `CopilotClient` の `mode` 設定: `copilot-cli`（全機能 ON、ローカル向け）vs `empty`（全機能 OFF、サーバー向け）
- 組み込みツールの `isolated` フィルタ: ホストマシンにアクセスできるツールを除外
- `virtualBash`: サードパーティライブラリ（Just Bash）を仮想ファイルシステム上で実行

**デモ**: TypeScript のサーバーサンプル（[Copilot SDK Server Sample](https://aka.ms/build26/BRK206)）で、セッション終了時にファイル自動削除 → 再接続時にバックエンドストレージから自動復元。github.com のチャット機能がこの方式で書き換え中であり、コスト削減にも寄与。

### Part 4: 今後のロードマップ（Patrick Nikoletich）

#### クロスサーフェス統一

VS Code・Visual Studio・github.com が同じエージェントハーネスに統一される。プラグイン・スキルの共有、ランタイム挙動の統一が 30–60 日以内に進行予定。

#### リモートクラウド環境

SDK にリモートクラウドセッション機能を追加。エージェントをクラウドサンドボックスにデプロイし、アプリ → ブラウザ → 別デバイスへのセッション移行が可能。アプリを閉じてもエージェントはクラウドで作業を継続。

#### Open Plugins

プラグイン・スキル・カスタムエージェント定義のパッケージ標準。どのサーフェス（CLI・IDE・Web）でも同じプラグインを共有するカタログを構築中。

#### パーソナライゼーション（メモリ）

User Preference Memory Service を GitHub 内でリリース。エージェントが記録したファクト・好みをサービス間で共有し、ある Copilot での体験が別の Copilot に引き継がれる。

#### Rust ランタイム書き換え

Node.js ランタイム（起動時 ~130MB）からの移行が進行中。期待される改善: 起動速度、メモリフットプリント、バンドルサイズ、対応プラットフォーム数。PoC として Wasm にコンパイル（~500KB）し、Chrome アドオンとして V8 サンドボックス内で動作 + リモートセッション接続を実演。

### デモで使用されたコードパターン

- `CopilotClient` → `Session` の作成パターン（C#）
- `replace` によるシステムプロンプトの `identity` セクション部分置換
- C# メソッドの `[Tool]` アトリビュートによるカスタムツール登録
- `mode: "empty"` プリセットによるサーバー向け最小権限構成（TypeScript）
- `SessionFS` インターフェースの `onDisk` / `inMemory` 実装
- Just Bash ライブラリによる仮想 bash ツール
